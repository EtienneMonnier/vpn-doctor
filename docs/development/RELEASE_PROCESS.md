# Release Process

## Versioning

Use semantic versioning:

```text
MAJOR.MINOR.PATCH
```

Initial versions:

- `0.1.0` MVP
- `0.2.0` secure profiles
- `0.3.0` diagnostics MVP
- `1.0.0` stable release

## Release checklist

- Tests pass.
- Changelog updated.
- Version bumped.
- Documentation updated.
- No secrets in repo.
- Packaging verified.
- Git tag created.

## Artifacts

Future release artifacts:

- source tarball
- RPM
- Flatpak
- screenshots
- AppStream metadata
